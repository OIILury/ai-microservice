ℹ️ Comment fonctionne cette application ?

▼

Architecture globale

Le frontend communique avec un microservice Python FastAPI tournant sur une carte embarquée Jetson Orin NX. Ce microservice transmet les requêtes à Ollama, un moteur d'inférence LLM qui exécute le modèle de langage localement. Aucune donnée ne quitte le réseau local.

Endpoint de correction (/api/corriger)

Le modèle reçoit un system prompt strict lui interdisant toute réponse conversationnelle. Il agit comme un filtre de transformation pure : entrée = texte brut, sortie = texte corrigé uniquement.

Endpoint de navigation (/api/trouver-page)

Le modèle reçoit une cartographie des pages disponibles et effectue une classification Zero-Shot — il détermine la page la plus pertinente sans avoir été entraîné spécifiquement pour ce site.

Confidentialité

L'architecture est "privacy-first" — le modèle LLM tourne intégralement en local sur le matériel Jetson, sans appel à des APIs cloud (pas d'OpenAI, pas d'Anthropic, pas de Google). Les données ne quittent jamais le réseau local.

Stack technique

  • Python 3.10+ / FastAPI / Pydantic / httpx
  • Ollama / Llama 3 8B
  • Jetson Orin NX / JetPack 6
